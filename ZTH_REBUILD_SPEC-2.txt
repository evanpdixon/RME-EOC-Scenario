ZERO TO HERO RADIO OPERATOR — SCENARIO SYSTEM
REBUILD SPECIFICATION
Radio Made Easy | Evan

This file contains everything needed to rebuild the entire ZTH scenario system
from scratch using Claude. Paste this file into a new Claude conversation and
say: "Rebuild the Zero to Hero scenario generator and brief/debrief guide from
this spec."

--------------------------------------------------------------------------------
SYSTEM OVERVIEW
--------------------------------------------------------------------------------

Two Python scripts generate all class materials via command line:

  zth_generator.py
    - Prompts for location config (3 modes: interactive / fill-in-blank / defaults)
    - Generates individual student task card PDFs (Student_01.pdf ... Student_N.pdf)
    - Generates individual student task card TXTs (Student_01.txt ... Student_N.txt)
    - Generates a Facilitator_Reference.pdf
    - Merges all PDFs into PrintAll_[Location].pdf
    - Generates AllTasks_[Location].txt (combined master text file)
    - Handles class sizes 12-30 with conflict-aware task distribution
    - PDF and TXT outputs are generated in the same pass — no extra steps

  generate_brief_debrief.py
    - No prompts; change LOCATION and VENUE at top of file
    - Generates ZTH_Brief_Debrief.pdf (3 parts: Brief, Debrief, Convoy)
    - Facilitator use only — not distributed to students

PDF stack: reportlab (pdfkit+wkhtmltopdf was considered; reportlab was chosen)
Merge: pypdf PdfWriter/PdfReader
No external API dependencies. All runs locally.

ZTH_GENERATOR.PY — FULL SPEC
--------------------------------------------------------------------------------

STARTUP PROMPT (printed at run):
  Mode 1 - Interactive   : prompts for each location detail
  Mode 2 - Fill-in-blank : prints blanks (_______________) where location
                           details go; asks only class_size and output_dir;
                           output files get _BLANK suffix
  Mode 3 - Defaults      : uses Jackson, Ohio values, no prompts

CONFIG FIELDS (all collected via get_config()):
  class_size    int     (default: 20)
  location      str     (default: "Jackson, Ohio")
  venue         str     (default: "86 Tick Ridge Rd, Jackson, OH")
  output_dir    str     (default: ".")
  main_road     str     road used for all neighbor addresses in A/C/D
                        (default: "Pattonsville Rd")
  flooded_road  str     road with flooding contradiction in E/L
                        (default: "Goose Run Rd")
  blocked_road  str     road with downed lines in K/B/N/J
                        (default: "E Broadway St")
  highway       str     nearest major highway (default: "US-35")
  main_st       str     local commercial corridor (default: "E Main St")
  addr_a        str     house number on main_road for Task A survivor
                        (default: "2847")
  addr_grandma  str     house number for Task C grandmother
                        (default: "2849")
  addr_uncle    str     house number for Task D uncle
                        (default: "2845")
  addr_neighbor str     house number for Task C neighbor-of-grandma
                        (default: "2851")
  grandma_name  str     (default: "Hazel Pitman")
  uncle_name    str     (default: "John")
  hospital      str     (default: "Jackson Area Medical Center / Holzer Jackson")
  gas_station   str     (default: "Marathon station on E Main St")
  local_store   str     (default: "Tractor Supply on E Main St")

DISTRIBUTION ALGORITHM:
  16 total tasks (A-P) in current build; designed to scale to 26 (A-Z) for
  classes up to 30 with 4 EOC students. Task N is the only priority=True task.
  - class_size <= total tasks: assign one task per student; leftover tasks
    absorbed as 2nd cards where conflict rules allow
  - class_size = total tasks: one task per student exactly
  - class_size > total tasks: all unique tasks assigned first; remaining
    students get duplicates cycling through the pool
  Conflict pairs are enforced — tasks that share critical intel cannot go
  to the same student. Conflicts are bidirectional.
  Use seed=42 for reproducible assignment.

PDF STYLING:
  All PDFs use reportlab SimpleDocTemplate, letter size, Helvetica fonts.
  Student cards: 0.85" margins all sides, 0.75" bottom for footer.
  Facilitator ref: 0.75" margins.
  Footer on every page: copyright line centered at 0.45" from bottom.
  COPYRIGHT = "© Radio Made Easy. Proprietary training material.
    Unauthorized reproduction, distribution, or use outside of
    licensed Radio Made Easy courses is strictly prohibited."

STUDENT CARD LAYOUT (per task):
  [If priority=True]: Red banner "⚠ HIGH PRIORITY — REPORT IMMEDIATELY TO NET CONTROL ⚠"
  Title: "Task [Letter] ([Phonetic])"  — red text if priority
  Subtitle: location string (grey, centered)
  Body: "You are within the disaster zone without cell phone, internet access,
         or electricity."
  Section: "Tasks to complete:" — bulleted list
  Section: "Information you can provide:" — bulleted list
  Footer right-aligned: [S-NN] student number reference
  If student has 2 tasks, PageBreak between them.

FACILITATOR REFERENCE LAYOUT:
  Header: location | class size | tasks distributed
  Red banner: high priority task callout for N
  Note about EOC routing around blocked_road for task N
  Section: "Built-in Contradictions" (4 pairs, see below)
  Section: "Cross-References" (6 pairs, see below)
  Section: "Student Assignments" grid table:
    columns: # | Code | Task(s) | Notes
    Notes column shows fac_notes from each task

PRINT-ALL:
  Facilitator reference first, then Student_01 through Student_N in order.
  Merged with pypdf.

================================================================================
PART 1 — SCENARIO BRIEF
================================================================================

Delivered to the full class before separating into groups and distributing cards.

SECTION: Overview
  Generic framing — Hurricane Helene or similar disaster. Two groups:
  Survivors and EOC. Both treat everything as real.

SECTION: The Scenario
  Major disaster has struck. For anyone inside the disaster zone:
  - No cellular service.
  - No internet access.
  - Possibly no ability to physically leave their location.
  - Their radio is their only lifeline.

  TIP BOX (talking point):
    "Put yourself in the shoes of a real survivor. You've just woken up after
    the storm. The power is out. Your phone has no signal. You can't drive out —
    roads may be blocked. The only thing connecting you to the outside world is
    this radio. That's where we're starting from."

SECTION: The Two Groups

  Survivors:
    - Go to vehicle, quiet area outside, anywhere away from EOC.
    - Task card contains your situation. You are that person.
    - Call into EOC on established net frequency.
    - Once your task is complete, you are NOT done (see below).

  EOC:
    - Stays in classroom. Full cell/internet access.
    - Establish net on GMRS simplex (channels 15–22).
    - Appoint Net Control to manage and prioritize traffic.
    - Support roles: researcher, logger, relay.
    - Thorough written log of every communication.
    - Connect survivor intel to other survivors (key emphasis).
    - Verify outside info before routing — online data may be outdated.
    - Always identify as training exercise to anyone outside scenario.

CRITICAL RED BANNER: "READ THIS TO THE CLASS VERBATIM OR VERY CLOSE TO IT"

  TIP BOX (near-verbatim read):
    "Here's something I need everyone to hear clearly. The information gathered
    during this exercise will be used in the next one. After we debrief, we are
    going to select one of these tasks and execute a real convoy supply run based
    solely on what was collected over the radio during this exercise.

    That means if a survivor passed bad information, or if the EOC gave wrong
    directions, or if anyone failed to document something they heard — that
    affects the convoy. Every piece of information shared on this net has real
    consequences in thirty minutes.

    A real survivor in a real disaster would be writing down everything they could
    possibly hear on that radio — not just what applies to them. Frequencies, road
    conditions, pharmacy hours, who's at what address — all of it. Treat this net
    as your only source of intelligence about the world outside your door.
    Document everything."

SECTION: After Your Task Is Complete
  - Continue monitoring the net — do not go quiet.
  - Document all information you hear from other survivors and the EOC.
  - You may be called on during debrief to contribute intel from what you heard.
  - In a real scenario, every survivor would be doing exactly this.

  TIP BOX:
    "Think of it like this — if the EOC tells another survivor that [flooded road]
    is flooded, that affects your route too. The net is a shared intelligence
    resource. Use it like one."

SECTION: Paper, Pen, and Legible Handwriting
  (Immediately before the pre-exercise checklist)
  Key points:
  - Every survivor needs something to write with and write on. Not optional.
  - In a real disaster your notes may be the ONLY surviving record of critical
    information — routes, resources, contacts, status of loved ones.
  - Write legibly. Notes only you can read are worthless to anyone helping you
    or taking over your situation.
  - Document EVERYTHING you hear on the net, not just what applies to your task.
  - Include specifics: addresses, phone numbers, hours, road names, directions,
    names. Vague notes are not actionable.
  - Your notes will be used in thirty minutes to plan the convoy. Write like
    your route depends on it — because it does.

  TIP BOX:
    "If the grid goes down and stays down, the information you wrote on that
    piece of paper may be the only intelligence you have. There's no cloud backup,
    no phone to check, no way to look it up again. Your notes are your database.
    Write clearly enough that someone else could act on them if you couldn't.
    That's the standard."

SECTION: Before You Send Them Out — Checklist (7-row grid table)
  Columns: checkbox | Item | Note
  Rows:
    EOC students identified (2–3) | Confirm before anyone leaves
    Net frequency established | Announce GMRS simplex channel to full class
    Task cards distributed | No one empty-handed
    Pen and paper confirmed | Every survivor needs something to write on
    Radios tested | Quick comms check before dispersing
    Timer / return signal set | Defined end time or agreed signal
    High-priority task noted | Task N (November) medical emergency — watch triage

TEACHING POINT 1: Survivor Stress vs. EOC Triage
  Core idea: Survivors feel genuine urgency about their task — realistic.
  Creates natural tension on a single shared channel.
  EOC must absorb that pressure without letting it dictate order of operations.
  EOC also balances: how much to research independently vs. wait for
  boots-on-ground intel from survivors who actually have eyes on conditions.

  Discussion questions:
    - Did Net Control hold the line, or did the most persistent caller win?
    - Was the medical emergency (Task N) recognized and elevated?
    - When fuel or road conditions asked about, did EOC look it up or wait
      for a survivor who had eyes on it?

  TIP BOX (two talking points):
    "Net Control's job isn't just calling on stations in order — it's making
    judgment calls under pressure. A survivor asking about propane feels just
    as urgent to them as someone calling in a medical emergency. Net Control has
    to hold both realities at once and still get the order right."

    "If someone asks about fuel availability and you Google it, you'll find
    hours and addresses — but none of that tells you whether the station has
    power, has gas in the ground, or is even open. The only person who can tell
    you that is someone who just drove past it. Learn when the internet helps and
    when you need to wait for a survivor report."

TEACHING POINT 2: Primary Sources Will Be Gone. Know Your Secondary/Tertiary.
  In a real disaster, the obvious first choice for any resource is almost always
  unavailable. Four specific examples:
    - Fuel: closest station may be out of power or gas. Know next 2 options.
      Which have generator backup?
    - Propane: exchange racks sell out within hours. Filling stations last longer.
      Know the nearest one.
    - Medical/Pharmacy: chain pharmacies overwhelmed or closed. Local
      independents often have stock when chains don't.
    - General rule: identify primary, secondary, and tertiary source for every
      critical resource NOW — before you need them.

  TIP BOX:
    "The homework I'm giving you isn't written down — it's mental. Drive your
    area. Know where the second gas station is. Know if the farm supply store
    carries propane. Know which pharmacy is locally owned. That knowledge is
    worth more than any piece of gear you own."

TEACHING POINT 3: Cash and Barter
  Several tasks returned cash-only results — realistic when power is out.
  Two things:
    - Cash in small bills: $100 bill useless if seller can't make change.
      Keep small denomination bills physically accessible, not in bank account.
    - Barter: in extended scenarios formal currency loses utility. Fuel, food,
      medical supplies, and labor are most common barter currencies in disasters.
      Think about what you have that others need.

TEACHING POINT 4: Routes and the Value of Local Intelligence
  Scenario had deliberate road obstacles that should have forced EOC to reroute.
  Directly previews the convoy.
  Discussion questions:
    - Did EOC incorporate road condition reports from survivors into directions?
    - When contradictory reports came in (E vs L on flooded road), did EOC
      resolve by recency?
    - The convoy runs on exactly this intel. Wrong routing = convoy hits obstacle.

  TIP BOX:
    "A map tells you where the road is. A survivor tells you whether you can
    actually drive it right now. There is no substitute for that. The person who
    drove that road 20 minutes ago is your most valuable intelligence asset —
    not Google Maps."

TEACHING POINT 5: Passing Messages Strategically
  Two real considerations most people haven't thought about:

  OPERATIONAL SECURITY:
    When you call the EOC to ask about a family member, you are broadcasting:
    - That a specific person lives at a specific address.
    - Whether they live alone.
    - Their age, health, and vulnerability.
    In a disaster with potential for looting or predatory behavior, that has
    real consequences. Think before you transmit.

  AUTHENTICATION — did the message actually get through?
    - Simple approach: tell EOC to ask your contact something only they'd know
      (what kind of car you drive, dog's name). Correct answer = message delivered.
    - Pre-planned approach: establish a family communication plan NOW — a code
      word or question/answer agreed on before any disaster. Faster, more secure.
    - Employer messages: use employee number or project reference — something
      your employer recognizes but a stranger wouldn't.

  TIP BOX:
    "Plan this before you need it. Sit down with your family and agree on a
    passphrase. Write it somewhere physical. If your radio is ever the only way
    to reach someone, you'll be glad you did."

PART 3 — CONVOY MISSION BRIEF
================================================================================

Red header banner. Delivered immediately after debrief.
Convoy is the capstone of Day 2 — everything applied in the real world.

SECTION: Selecting the Mission
  Default target: Task P (Papa) fuel run to gas station within ~5 miles.
  Rules:
  - Use only information gathered during the scenario — no new research.
  - If contradictory intel exists, resolve with class before departure.
  - All inter-vehicle communication uses radio only.

  TIP BOX:
    "If class documentation was poor and you don't have a confirmed destination,
    use it as a teaching moment: 'This is why we document. We can't execute a
    mission on information we don't have.'"

SECTION: Convoy Brief Checklist (7-row grid table)
  Columns: checkbox | Item | Detail
  Rows:
    Objective  | State mission in one sentence: "We are conducting a convoy
                 supply run to [LOCATION] based on intelligence gathered during
                 the scenario."
    Route      | Brief planned route including known obstacles and alternates.
                 Everyone confirms written down before moving.
    Frequency  | Confirm convoy channel. All vehicles monitor and transmit
                 on this frequency only.
    Vehicle order | Assign lead, trail, middle. Lead does not outpace convoy.
    Rally point | Designate regroup location if comms lost or vehicle has problem.
    Rules       | No vehicle breaks from convoy without announcing on net.
                  Net Control maintains accountability of all vehicles at all times.
    Return plan | State how and when group returns to venue; how course wraps up.

SECTION: During the Convoy
  - Net Control makes periodic check-ins — keep the net active.
  - Call out landmarks and turns over radio as you approach them.
  - Route obstacle encountered: Net Control calls all-stop, convoy holds,
    route decision made over the net before anyone moves.
  - Maintain radio discipline — live application of everything taught.

SECTION: Wrap-Up After the Convoy
  - Brief debrief in parking lot or classroom — 5 minutes.
  - "What worked? What would you change?"
  - Tie it back: quality of the intel determined quality of the mission.
  - Transition to final manual pages and close out the course.

DESIGN DECISIONS & NOTES FOR REBUILD
================================================================================

TEXT EXPORT (Signal / copy-paste):
  Every run generates text output alongside PDFs automatically — no separate
  step or flag needed.

  OUTPUT FILES:
    Student_NN.txt        — one file per student, that student's task(s) only
    AllTasks_[Loc].txt    — all students combined in one file, sequential order
    (fill-in-blank mode appends _BLANK to both, same as PDFs)

  FORMAT RULES:
    - "=" * 48 as major separator between cards
    - "-" * 48 as minor separator (not currently used but available)
    - *asterisks* around bold labels (renders natively in Signal)
    - "- " prefix for all bullet items
    - "!! CAPS !!" for priority banner (no color in text, caps stand in)
    - [S-NN] student reference at bottom of each card
    - Double newline between cards in master file

  INTENDED WORKFLOW:
    Individual .txt files: copy entire file, paste directly into Signal DM
      before that student leaves the room.
    Master .txt file: keep as a reference or use to grab individual sections
      on the fly if a student loses their card.
    Facilitator notes are NOT included in text output — student-facing content
      only. Fac notes stay in Facilitator_Reference.pdf only.

  FUNCTIONS:
    task_card_text(task, student_num, cfg) -> str
      Formats a single task as a Signal-ready text block.

    build_student_txt(student_num, tasks, cfg, output_dir) -> path
      Writes one student's card(s) to Student_NN.txt.

    build_master_txt(assignments, cfg, output_dir, blank_tag="") -> path
      Writes all students to AllTasks_[Location][blank_tag].txt.
      Master file begins with a header block showing location, class size,
      and total tasks distributed.

WHY REPORTLAB:
  pdfkit+wkhtmltopdf was evaluated but had system dependency issues. reportlab
  is pure Python, reliable, and produces consistent output. All PDFs use
  SimpleDocTemplate with onFirstPage/onLaterPages footer callbacks.

WHY SEED=42:
  Assignment distribution uses random.seed(42) so the facilitator reference
  and student PDFs always match on re-runs. Change the seed if you want a
  different distribution.

FILL-IN-BLANK MODE:
  All location-specific cfg values are replaced with BLANK = "_______________"
  before tasks are built. Output files get _BLANK suffix so they don't
  overwrite real location runs.

CONFLICT SYSTEM:
  Conflicts are defined per-task as a list of letters. Enforcement is
  bidirectional — if A conflicts with C, C also cannot go to A's student.
  The distribute() function checks both directions.

TASK ORDER IN PRINT-ALL:
  Facilitator reference is always first, then students in order (S-01 to S-N).
  This makes it easy to cut the facilitator pages off the front before
  distributing.

EOC COMPOSITION:
  For classes of 16-22: 1-2 students handle EOC, rest are survivors.
  For classes of 23-30: up to 4 students handle EOC, leaving up to 26
  survivors each with a unique task (A-Z). The facilitator pulls EOC students
  by name before distributing cards — they do not get a task card.
  EOC composition is a logistical decision made at runtime, not automated.

BRIEF/DEBRIEF SCRIPT:
  generate_brief_debrief.py is location-agnostic except for LOCATION and VENUE
  at the top. The debrief teaching points do NOT reference specific road names —
  they are kept generic so the same script works for any location with no edits
  beyond the two variables. The task reference table in the debrief does contain
  the Jackson road names as illustrative examples; those should be updated
  manually for new locations or left generic.
